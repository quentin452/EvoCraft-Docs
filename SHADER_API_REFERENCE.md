## 1. File Format

`.evo_shader` files are written in **Rust-like syntax** (not GLSL). The `evo-shader-compiler` front-end parses them, resolves attribute macros, injects push-constant layouts and GBT bindings, then compiles to SPIR-V via `rust-gpu`.

Key syntactic rules:

- Standard Rust items: `use`, `const`, `struct`, `fn`, `impl`, `type`.
- Shader entry points are **free functions** annotated with `#[evo_shader]` or `#[spirv(...)]`.
- Push constants are plain `#[repr(C)]` structs; a parameter named `pc` or `push` is auto-decorated.
- Buffer Device Address (BDA) buffers are accessed as `&PhysicalStorageBuffer<[T]>` parameters.
- `glam` types (`UVec3`, `Vec3`, `Vec4`, `Mat4`) are available; `spirv_std` intrinsics via `use`.
- `use shared::{...}` imports from `projects/libs/shared` (compiled into every shader).

```rust
//! my_compute.evo_shader

use shared::{EnginePushConstants, GBT_SLOT_ENTITY_CORE};
use glam::UVec3;

#[repr(C)]
#[derive(Copy, Clone)]
struct PushConstants {
    entity_count: u32,
    delta_time: u32, // f32 bits -- use f32::from_bits()
}

#[evo_shader(preset = "EntityReadWrite", threads = 64)]
fn my_compute(
    global_id: UVec3,                // auto-decorated: GlobalInvocationId
    my_buffer: &mut [u32],           // auto-decorated: storage_buffer binding
    pc: &PushConstants,              // auto-decorated: push_constant
    engine_pc: &EnginePushConstants, // auto-decorated: engine push constants
) {
    let idx = global_id.x as usize;
    if idx >= pc.entity_count as usize {
        return;
    }
    // ...
}
```

## 2. High-Level Style -- `#[evo_shader]`

The preferred authoring style. The compiler selects a **preset** that provides canonical GBT binding slots, default workgroup sizes, and optional prologue/epilogue injection.

```rust
#[evo_shader(
    preset = "EntityReadWrite",  // binding layout preset (see section 5)
    threads = 64,               // workgroup X size (or threads(x,y,z))
    entry_point_name = "tick",  // override SPIR-V entry point name
    priority = 10,              // dispatch order within category
    worldgen_queue = "terrain", // enables worldgen prologue (see section 8)
    signal_completion = true,   // enables worldgen epilogue (see section 8)
)]
fn my_shader(global_id: UVec3, /* ... */) { /* ... */ }
```

### `#[evo_shader]` fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `preset` | `&str` | `""` (none) | Preset name selecting a canonical binding layout. |
| `threads` | `u32` or `(x,y,z)` | preset default | Workgroup size. Single integer sets X; tuple sets X/Y/Z. |
| `entry_point_name` | `&str` | fn name | Override the SPIR-V entry-point symbol. |
| `priority` | `i32` | `0` | Dispatch ordering within the category -- lower runs first. |
| `worldgen_queue` | `&str` | `""` | Worldgen queue name; injects pop-from-queue prologue. |
| `signal_completion` | `bool` | `false` | Injects completion-signal epilogue (used with `worldgen_queue`). |

### Auto-wired parameter names

| Parameter name | Auto decoration applied |
|---------------|-------------------------|
| `global_id` | `#[spirv(global_invocation_id)]` |

## 3. Low-Level Style -- `#[spirv]`

Used for render pipelines (vertex/fragment/mesh/task) and BDA-only compute shaders that do not fit any preset. All decorations are explicit.

```rust
// Task + Mesh + Fragment pipeline example
#[spirv(task_ext(threads(32, 1, 1)))]
pub fn my_task(
    #[spirv(global_invocation_id)] global_id: UVec3,
    #[spirv(task_payload_workgroup_ext)] payload: &mut MyPayload,
    #[spirv(push_constant)] pc: &MyPush,
) { /* cull, emit mesh workgroups */ }

#[spirv(mesh_ext(
    threads(64, 1, 1),
    output_vertices = 64,
    output_primitives = 64,
    output_triangles,
))]
pub fn my_mesh(
    #[spirv(global_invocation_id)] global_id: UVec3,
    #[spirv(task_payload_workgroup_ext)] payload: &MyPayload,
    #[spirv(descriptor_set = 0, binding = 0)] vertices: &[Vertex],
    out_vertices: &mut Output<[Vec4]>,
    out_primitives: &mut Output<[Triangle]>,
) { /* emit vertices + primitives */ }

#[spirv(fragment)]
pub fn my_frag(
    #[spirv(flat, location = 0)] color: Vec4,
    out_color: &mut Vec4,
) { *out_color = color; }
```

## 8. GPU Worldgen Pattern

Worldgen shaders use a queue-based dispatch pattern where the GPU self-schedules generation tasks. Use `worldgen_queue` and `signal_completion` in `#[evo_shader]` to opt in.

The compiler injects:

- **Prologue**: pop a chunk coordinate from the named worldgen queue. If the queue is empty the thread exits immediately.
- **Epilogue**: atomically signal completion so the CPU can advance the streaming state machine.

use shared::EnginePushConstants;
use glam::UVec3;

    threads(16, 16, 1),
