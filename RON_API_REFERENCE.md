## How Attributes Work

EvoCraft mods are written in `.evo_mod` files using Rust-like attribute syntax:

```rust
#[evo_block(
    name = "DIAMOND_ORE",
    texture = "diamond_ore",
    light_emission = 2,
)]
```

The compiler parses each `#[evo_*(...)]` attribute into a typed Rust struct and collects
them into the mod's manifest. At runtime the engine reads the manifest to register blocks,
services, dispatches, etc.

**Conventions:**
- Fields marked `(default: X)` can be omitted; the default value is used.
- Fields marked `(required)` must be provided.
- `Option<T>` fields default to `None` when omitted.
- `Vec<T>` fields default to `[]` when omitted.

### evo_gpu_pipeline

Declares a self-contained GPU compute pipeline bundling its own buffers, services, and a dispatch schedule. Aliases: `evo_compute`, `evo_pipeline`. Useful for encapsulated subsystems (e.g. a physics pipeline, a meshing pipeline).

```rust
#[evo_gpu_pipeline(
    name = "particle_sim",
    buffers = [
        (name = "particles",      usage = Storage, size = "65536"),
        (name = "particle_count", usage = Counter),
    ],
    services = [
        ComputeService(name = "sim_step",  shader = "particle_sim_comp"),
        ComputeService(name = "emit_step", shader = "particle_emit_comp"),
    ],
    dispatch = Batch(
        trigger = EveryFrame,
        passes = [
            Pass(service = "emit_step",  threads = Explicit(64, 1, 1)),
            Barrier,
            Pass(service = "sim_step",   threads = Explicit(256, 1, 1)),
        ],
    ),
)]
```

**Fields:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `String` | required | Unique pipeline name. Namespaces its services and buffers. |
| `buffers` | `Vec<PipelineBufferDef>` | `[]` | Pipeline-owned buffers. Each has `name`, `usage` (`Storage`, `Counter`, `Indirect`, `Upload`, `CpuVisible`), `size`/`element_size`/`count`, `init`, `cpu_read`, `cpu_write`. |
| `services` | `Vec<ServiceDef>` | `[]` | Compute or graphics services within this pipeline (same syntax as `#[evo_service]`). |
| `dispatch` | `Option<PipelineDispatchDef>` | `None` | `Single(service, trigger, threads, condition, push_constants)` or `Batch(trigger, passes)`. |
| `readback` | `Option<PipelineReadbackDef>` | `None` | GPU to CPU readback: `buffer`, `size`, `trigger` (`EveryFrame`, `EveryNFrames(n)`, `OnDemand`). |

### evo_dispatch

Declares when and how a named GPU service is dispatched. Multiple `#[evo_dispatch]` attributes can appear in the same file to schedule different services in different phases.

```rust
#[evo_dispatch(Dispatch(
    service = "physics_tick",
    trigger = FixedUpdate,
    phase = PhysicsTick,
    condition = GameState("Playing"),
    threads = AutoFromReflection("entity_count"),
    push_constants = [
        Auto("delta_time_fixed"),
        Auto("entity_count"),
    ],
    reads  = ["entity_state"],
    writes = ["entity_state"],
    after  = ["mod-physics::integrate"],
))]
```

**Variants:** `Dispatch`, `BatchDispatch`, `BlockTickerDispatch`, `ComputeSystemDispatch`

**Fields for `Dispatch`:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `service` | `String` | required | Name of the service to dispatch. |
| `trigger` | `DispatchTrigger` | `EveryFrame` | `EveryFrame`, `FixedUpdate`, `OnEvent("name")`. |
| `threads` | `ThreadsDef` | `AutoFromReflection("entity_count")` | `Explicit(x,y,z)`, `Auto("buf",y,z)`, or `AutoFromReflection("name")`. |
| `condition` | `Option<ConditionDef>` | `None` | `GameState("name")`, `Flag("name")`, `SessionFlag("name")`, `And([...])`, `Or([...])`, `Not(...)`, `WeatherActive(f32)`. |
| `push_constants` | `Vec<PushConstantValue>` | `[]` | `Auto("name")`, `Const(f32)`, `ConstU32(u32)`, `Config("var")`, `FlagMask("flag")`. |
| `reads` | `Vec<String>` | `[]` | Buffers read â€” for automatic task-graph dependency resolution. |
| `writes` | `Vec<String>` | `[]` | Buffers written â€” for automatic task-graph dependency resolution. |
| `after` | `Vec<String>` | `[]` | Qualified dispatch names that must complete before this one. |
| `before` | `Vec<String>` | `[]` | Qualified dispatch names that must run after this one. |
| `phase` | `FramePhase` | `EarlyUpdate` | `EarlyUpdate`, `PhysicsTick`, `PostPhysics`, `PreRender`, `Render`, `PostProcess`, `LateUpdate`, `Headless`. |

**Known auto push constant names:** `delta_time`, `delta_time_fixed`, `frame_number`, `entity_count`, `game_state`, `time`, `camera_x/y/z`, `chunk_height`, `weather_*`, `screen_width/height`.

### evo_buffer

Declares a named GPU buffer. Registered in the mod's runtime resource registry and bindable to services by name â€” either explicitly via `#[evo_binding]` or automatically via SPIR-V reflection.

    name = "player_state",
    type = Storage,
    size = "1024",

    init = Zeroed,

// Element-count form (size = element_size * count)

    name = "entity_transforms",
    type = Storage,
    element_size = 64,
    count = 4096,

**Fields:**

### evo_binding

Declares an explicit descriptor binding between a named service and a named buffer. Without this attribute the engine auto-binds by matching shader binding names to registered buffer names via SPIR-V reflection. Use `#[evo_binding]` to override auto-binding or target an engine-owned buffer.

```rust
// Bind a mod buffer
#[evo_binding(
    service = "my_compute",
    buffer = "my_buffer",
    slot = 0,
    access = ReadOnly,
)]

// Bind an engine-managed buffer
#[evo_binding(
    service = "my_compute",
    engine_buffer = "block_data",
    slot = 4,
    access = ReadWrite,
)]
```

**Fields:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `service` | `String` | required | Name of the service to bind to. |
| `buffer` | `Option<String>` | `None` | Name of a mod-declared buffer (from `#[evo_buffer]`). |
| `engine_buffer` | `Option<String>` | `None` | Name of an engine-owned buffer (e.g. `"block_data"`, `"chunk_map"`). |
| `slot` | `Option<u32>` | `None` | Explicit descriptor binding index. If omitted, resolved by SPIR-V reflection. |
| `access` | `Option<AccessType>` | `ReadWrite` | `ReadOnly`, `WriteOnly`, or `ReadWrite`. |

### evo_temp_buffer_pool

Declares a pool of reusable temporary GPU buffers for multi-pass compute pipelines. The engine allocates/recycles buffer sets matching the slot definitions at dispatch time, avoiding per-dispatch allocation overhead. Referenced by `#[evo_dispatch_sequence]`.

```rust

    name = "meshing_temp_pool",
    min_size = 65536,
    growth = PowerOfTwo,
    slots = [
        (name = "voxels",     usage = [Storage],                        memory = GpuOnly,  size_expr = "chunk_volume * 4"),
        (name = "mask",       mirror_of = "voxels"),
        (name = "quads",      usage = [Storage],                        memory = GpuOnly,  size_expr = "16384 * 32"),
        (name = "quad_count", usage = [Storage, TransferDst, TransferSrc], memory = GpuOnly, size = 4),
        (name = "indirect",   usage = [Storage, Indirect, TransferDst], memory = GpuOnly,  size = 12),
        (name = "staging",    usage = [TransferDst],                    memory = GpuToCpu, size = 8),
    ],
)]
```

**Fields:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `String` | required | Pool name â€” referenced in `#[evo_dispatch_sequence]` via `temp_pool`. |
| `min_size` | `u64` | `65536` | Minimum buffer size â€” smaller allocations are rounded up. |
| `growth` | `GrowthStrategy` | `PowerOfTwo` | `PowerOfTwo` or `Exact`. |
| `slots` | `Vec<TempBufferSlotDef>` | `[]` | Buffer slots: `name`, `usage` (`[Storage, TransferSrc, TransferDst, Indirect]`), `memory` (`GpuOnly`, `GpuToCpu`, `CpuToGpu`), `size` (fixed bytes) or `size_expr` (variables: `chunk_volume`, `chunk_width`, `chunk_height`), `mirror_of`. |

### evo_dispatch_sequence

Declares a data-driven multi-pass GPU compute sequence tying together a pipeline's services with a temp buffer pool. Replaces hardcoded Rust orchestration code.

```rust
#[evo_dispatch_sequence(
    name = "greedy_meshing",
    pipeline = "greedy_meshing_pipeline",
    temp_pool = "meshing_temp_pool",

        (slot = "quad_count", size = 4,  value = 0),
        (slot = "indirect",   size = 12, value = 0),
    ],

        Dispatch(service = "greedy_mask", workgroups = Expr("sx", "sy", "sz"),
                 push_constants = Full,
                 bindings = [(binding = "_voxels", source = TempSlot("voxels"))]),
        Barrier,
        Loop(count = 3, variable = "axis", steps = [
            Dispatch(service = "greedy_quads", workgroups = Fixed(64, 1, 1),
                     push_constants = WithOverrides([(field = "axis", value = LoopVariable)])),
        ]),
        Dispatch(service = "emit_quads", workgroups = Indirect(slot = "indirect", offset = 0),
                 push_constants = Full),
    ],
)]
```

**Fields:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `String` | required | Unique sequence name. |
| `pipeline` | `String` | required | Name of the `#[evo_gpu_pipeline]` that owns the services. |
| `temp_pool` | `Option<String>` | `None` | Name of the `#[evo_temp_buffer_pool]` providing scratch buffers. |
| `pre_fills` | `Vec<SequencePreFillDef>` | `[]` | Zero/constant-fill commands before dispatches: `slot`, `offset`, `size`, `value`. |
| `steps` | `Vec<SequenceStepDef>` | required | Ordered steps: `Dispatch { service, workgroups, push_constants, bindings }`, `Loop { count, variable, steps }`, `Barrier`. |

### evo_block

Defines a voxel block type. Each `#[evo_block]` registers one entry in the block registry and uploads its parameters to the GPU block table. The `color` shorthand sets `tint_r/g/b/a`.

```rust
#[evo_block(
    name = "GLOWING_STONE",
    texture = "glowing_stone",
    density = 1.5,
    transparent = false,
    light_emission = 8,
    breakable = true,
    break_sound = "stone_break",
    behavior = Storage,
)]

// Color shorthand
#[evo_block(name = "TINTED_GLASS", texture = "glass", transparent = true,
            color = (0.2, 0.6, 1.0, 0.5))]
```

**Fields:**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `String` | required | Unique block name (ALL_CAPS convention). |
| `id` | `Option<u32>` | `None` | Explicit stable block ID (0 is reserved for air). |
| `texture` | `Option<String>` | `None` | Default texture for all faces. |
| `texture_top` | `Option<String>` | `None` | Top face texture override. |
| `texture_side` | `Option<String>` | `None` | Side faces texture override. |
| `texture_bottom` | `Option<String>` | `None` | Bottom face texture override. |
| `texture_namespace` | `Option<String>` | `None` | Override texture namespace for cross-mod textures. |
| `density` | `f32` | `1.0` | Block density (affects physics interactions). |
| `flow` | `f32` | `0.0` | Liquid flow speed (non-zero enables liquid physics). |
| `transparent` | `bool` | `false` | If true, block does not occlude neighbors. |
| `light_emission` | `u8` | `0` | Light level emitted (0..15). |
| `color` | `Option<[f32; 4]>` | `None` | Shorthand for `tint_r/g/b/a`. Wins over individual tint fields. |
